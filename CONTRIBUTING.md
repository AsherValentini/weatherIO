# Contributing to WeatherIO

Thank you for your interest in contributing to WeatherIO! While currently just an exposition project I welcome contributions to make this project better. Please follow the guidelines below to contribute effectively.

## How to Contribute

### Reporting Bugs
- Ensure the bug hasn't already been reported by searching the [issues](https://github.com/weatherIO/issues).
- Provide detailed information, including:
  - Steps to reproduce the bug
  - Expected behavior
  - Actual behavior
  - Environment details (OS, Python version, etc.)

### Suggesting Features
- Open a new issue with the "Feature Request" label.
- Describe the feature and its benefits clearly.

### Submitting Changes
1. **Fork the Repository**:
   Click the "Fork" button on the [GitHub repository](https://github.com/your-repo).

2. **Clone the Repository**:
   ```bash
   git clone https://github.com/your-username/weatherIO.git

3. **Create a Branch**:
   git checkout -b feature/your-feature-name

4. **Make Your Changes**:
   -Follow the project's codng standards.
   -Add or update test for your changes.

5. **Run Test: Ensure all tests pass**:
   pytest tests/

6. **Commit Changes: Write clear, concise commit message:
   git add .
   git commit -m "add your detailed message here"

7. **Push Changes**:
   git push origin feature/your-feature-name

8. **Open a Pull Request**:
   -Provide a summary of the changes
   -Any relavent issue number
   -Steps to test the changes (please)

### Style Guidelines
    - Follow PEP 8 for Python code.
    - Use descriptive names for variables, functions, and classes
    - Inlcude docstrings and comments for readablity
